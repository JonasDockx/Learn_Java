Abstraction

Abstract classes, interfaces, and when to use which. Print this.

Abstract class

public abstract class Animal {
    private String name;
    public Animal(String name) { this.name = name; }
    public String getName() { return name; }   // finished method
    public abstract String speak();             // no body — must override
}
  • Cannot be instantiated (new Animal() won't compile).
  • May mix finished methods + state with abstract (unfinished) methods.
  • Subclasses must implement every abstract method.

Interface

public interface Trainable {
    String learnTrick(String trick);   // implicitly public abstract
}

public class Dog extends Animal implements Trainable {
    @Override public String speak() { return getName() + " woof"; }
    @Override public String learnTrick(String t) { return getName() + " did " + t; }
}

One-vs-many

extends one class · implements many interfaces. Inheritance shares identity; interfaces share ability.

is-a vs has-a (composition)

class Dog extends Animal { }     // is-a   → inheritance
class Car { private Engine engine; }   // has-a → composition (often preferred)

Favour composition unless there's a real is-a with behaviour to share polymorphically.
