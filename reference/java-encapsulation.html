Encapsulation & Access

Private data, controlled access. Print this.

The pattern

public class Dog {
    private int age;                 // 1. hide the field

    public int getAge() {            // 2. controlled read
        return age;
    }

    public void setAge(int age) {    // 3. controlled write + validation
        if (age < 0) throw new IllegalArgumentException("age < 0");
        this.age = age;
    }
}

Default for a junior: fields private, methods public unless you have a reason otherwise.

Naming conventions

  • Getter: getX() — or isX() for a boolean.
  • Setter: setX(value), returns void, validates before assigning.
  • Construct through the setter so validation runs at creation too.

Why bother

Encapsulation = data + the rules that protect it, bundled, with a guarded door. private is compiler-enforced, so invalid state becomes impossible, not just discouraged.
