Spring: IoC & Dependency Injection

Spring creates and wires your objects. Print this.

Vocabulary

TermMeans
IoC containerSpring's manager that builds + wires your objects
Beanan object the container creates and manages
Dependency injectionthe container passing a bean's dependencies in
Component scanningSpring finding your annotated classes to register

Core annotations

@SpringBootApplication      // entry point; starts container, scans package
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

@Component                   // "manage a bean of this class"
public class EmailSender implements MessageSender { ... }

@Service                     // @Component that means "business logic"
public class NotificationService {
    private final MessageSender sender;
    public NotificationService(MessageSender sender) {   // injected — no annotation needed
        this.sender = sender;
    }
}

How wiring happens

start  →  scan package for @Component/@Service  →
  create one bean each  →  read each constructor  →  inject matching beans

No new in your code. If two beans fit one type, mark one @Primary.

Packages (required)

package com.jonas.demo;   // top of every file
// folder path must match:  src/main/java/com/jonas/demo/

Spring scans the entry class's package and below — so real apps use packages, not the default one.

Run it

cd spring-demo
mvn spring-boot:run       # start the app (Ctrl+C to stop)

The big picture

You declare what exists (@Component) and what it needs (constructor params); Spring works out how to build and connect it. It's the Dependency Inversion Principle, automated.
