Inheritance & Polymorphism

Sharing behaviour and the one-type-many-forms payoff. Print this.

The shape

public class Animal {
    private String name;
    public Animal(String name) { this.name = name; }
    public String getName() { return name; }
    public String speak() { return name + " makes a sound"; }
}

public class Dog extends Animal {
    public Dog(String name) { super(name); }   // call parent constructor
    @Override
    public String speak() { return getName() + " says woof"; }
}

Keywords

KeywordMeans
extends Animalthis class is-a Animal; inherits its members
super(...)call the parent constructor (first line of yours)
super.method()call the parent's version of a method
@Override"I'm replacing a parent method" — compiler-checked

Polymorphism

Animal a = new Dog("Rex");   // declared Animal, really a Dog
a.speak();                   // "Rex says woof" — object's type wins

Animal[] zoo = { new Dog("Rex"), new Cat("Mittens") };
for (Animal x : zoo) System.out.println(x.speak());  // each its own way

Declared type = what the compiler lets you call. Object type = whose method actually runs (decided at runtime).

Python → Java

class Dog(Animal): → class Dog extends Animal
super().__init__(name) → super(name)
duck typing → polymorphism via a shared supertype the compiler checks

Override vs overload (don't mix up)

  • Override: same method signature, in a subclass — replaces behaviour.
  • Overload: same method name, different parameters, same class — e.g. print(int) and print(String).
