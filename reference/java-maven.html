Maven

Project layout, pom.xml, and the commands. Print this.

Standard layout

my-project/
├── pom.xml              configuration
├── src/main/java/       production code
├── src/test/java/       test code
└── target/              build output (gitignored)

pom.xml essentials

<groupId>com.jonas</groupId>        who owns it
<artifactId>hello-maven</artifactId> project name
<version>1.0</version>               project version

<dependencies>                       libraries Maven downloads
  <dependency>
    <groupId>...</groupId>
    <artifactId>...</artifactId>
    <version>...</version>
    <scope>test</scope>              (optional) only for tests
  </dependency>
</dependencies>

A dependency's groupId:artifactId:version are its coordinates — its unique address in the Maven Central repository.

Commands

CommandDoes
mvn compilecompile main code
mvn testcompile + run tests
mvn packagebuild a .jar into target/
mvn cleandelete target/
mvn clean testfresh build, then test

Run from the folder containing pom.xml. Downloaded libraries cache in ~/.m2/.

Reading the result

Tests run: 1, Failures: 0, Errors: 0   → all good
BUILD SUCCESS                          → compiled + tests passed
BUILD FAILURE                          → something broke (read upward)

Python comparison

Maven ≈ pip + a project/build standard. pom.xml ≈ requirements.txt + pyproject.toml, but Maven also fixes the folder layout and runs the build/test lifecycle.
