Key Terms, Plainly

The vocabulary the OO lessons are built from. Each with a tiny example. Print this.

Body

The code inside a method's { } — the steps it actually runs.

String turnOn() { return "Lamp is on"; }   // body = return "Lamp is on";
String turnOn();                            // NO body — ends in ; , no steps

(Method) signature

A method's identity: its name + the inputs (parameters) it takes. How to call it, not what it does. The part before the {.

turnOn()          // name "turnOn", no inputs
setAge(int age)   // name "setAge", one input of type int

State

The data an object currently holds — the values in its fields.

Dog d = new Dog("Rex", 3);   // state: name = "Rex", age = 3
d.setAge(4);                 // changes state to age = 4

"No state" = no fields to hold data (true of an interface).

To implement

Two related meanings:

  • implement a method = write its body (the real code).
  • implements (keyword) = a class promising to provide every method an interface lists.
class Lamp implements Switchable {   // promises to fulfil Switchable
    public String turnOn() { ... }   // ...by implementing turnOn()
}

Contract

A promise that certain methods exist, without saying how they work. A concept, not a keyword. In Java, an interface is a contract: it lists the promises; an implementing class must deliver them.

Abstract class

A class you cannot build with new; it exists only to be a parent. Can mix finished methods (with bodies) and abstract methods (no body) that subclasses must finish. A half-built blueprint.

abstract class Vehicle {     // can't do new Vehicle()
    abstract String sound(); // subclasses must finish it
}

Interface

A pure contract: only signatures — no bodies, no state. A class implements it and must write all the methods. A class can implement many.

interface Switchable { String turnOn(); }   // just the promise

In one line

A signature is the promise · a body is the fulfilment · state is the data an object holds · to implement is to fulfil · a contract is a set of promises · an interface is a contract in code · an abstract class is a half-built parent.
