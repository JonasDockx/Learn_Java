Clean Code

Habits that make code readable. Print this.

The three highest-impact habits

HabitDo
Intention-revealing namesapplyDiscount(price, percent), not c(p, d)
No magic numbersMINIMUM_AGE, not a bare 18
Small, single-purpose methodsone method = one sentence

Constants with final

private static final int MINIMUM_AGE = 18;   // can't be reassigned

final = unchangeable. static final = one shared constant for the class. Named in UPPER_SNAKE_CASE.

Comments

Good code rarely needs comments to explain what. Comment the why — a non-obvious decision or workaround. A comment restating the code is noise; a clear name is better.

The mindset

Optimise for the reader, not the writer. You will read this code far more often than you wrote it. Refactoring (Lesson 24) is how you move messy code toward clean — safely, with tests green.
