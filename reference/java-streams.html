Streams

Process a collection as a pipeline. Print this.

The pipeline

var result = nums.stream()        // source
    .filter(n -> n % 2 == 0)       // intermediate: keep
    .map(n -> n * 10)              // intermediate: transform
    .toList();                     // terminal: produce result

Three kinds of step

StepRole
.stream()start a pipeline from a collection
.filter(lambda)keep items where the lambda is true
.map(lambda)transform each item
.sorted()(intermediate) sort the items
.toList()(terminal) gather into a List
.forEach(lambda)(terminal) do something with each
.count()(terminal) how many items

Intermediate ops chain and are lazy; nothing runs until a terminal op. The source collection is never modified.

Python comparison

# comprehension
[n*10 for n in nums if n % 2 == 0]

// stream
nums.stream().filter(n -> n%2==0).map(n -> n*10).toList()

Remember

Source → keep (filter) → transform (map) → gather (toList/forEach). Every filter/map takes a lambda. The original list stays unchanged.
