CI/CD

Automated build + test on every push. Print this.

The terms

TermMeans
CI (Continuous Integration)auto build + test on every push
CD (Continuous Delivery/Deployment)auto package + ship after tests pass
Workflowthe YAML file describing the steps
Runnerthe machine GitHub runs it on

Minimal GitHub Actions workflow (Java + Maven)

# .github/workflows/ci.yml
name: CI
on:
  push:
  pull_request:
jobs:
  build-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '21'
      - run: mvn -B test
        working-directory: hello-maven

The mental model

push  →  GitHub spins up a fresh machine  →
  checkout code  →  install Java  →  mvn test  →  green ✓ or red ✗

It's your local steps, automated and run for everyone on every change.

YAML

Indentation-based config (like Python — spaces, not braces). Keys end in :; nesting is by indentation. A misaligned space breaks it — check indentation first when a workflow won't parse.

Where it lives

Repo root: .github/workflows/*.yml. Results appear under the repo's Actions tab. Red X = a step failed (usually a test); click in to read the log.
