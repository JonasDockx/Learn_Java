Functional Interfaces & Lambdas

A lambda implements a one-method interface, inline. Print this.

Functional interface

interface Greeter {           // exactly ONE abstract method
    String greet(String name);
}

Lambda syntax

Greeter g = name -> "Hi " + name;
//          └params┘ └arrow┘ └result┘

Adder sum = (a, b) -> a + b;        // 2+ params: parentheses
Runnable r = () -> doSomething();    // 0 params: empty ()
// multi-line body uses braces + return:
Greeter g2 = name -> {
    String upper = name.toUpperCase();
    return "HI " + upper;
};

Calling it

g.greet("Jonas");     // call the single method, as normal

The rule

Left of -> = parameters · right = result. The lambda's type is a functional interface; Java infers the method and parameter types from it, so you write only the logic. A lambda is always an implementation of some one-method interface — Java has no standalone functions.

Python → Java

Why it matters

Lambdas are the input to streams (filter, map, …) and appear all over modern Java (comparators, event handlers, Spring callbacks).
