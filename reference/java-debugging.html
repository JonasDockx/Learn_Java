Debugging

Find a bug calmly and repeatably. Print this.

The loop

  1. Reproduce — run it; get the failure reliably.
  2. Read the stack trace — top line in your file = where; exception + message = what.
  3. Inspect — print suspect values just before the failing line.
  4. Fix & verify — change one thing; run again.

Reading a stack trace

Exception ... NullPointerException: ... is null
    at BugHunt.main(BugHunt.java:9)   ← top with YOUR file = the spot

Scan for the first line naming your .java file. The message says what went wrong.

Common exceptions

ExceptionUsually means
NullPointerExceptionused something that was null
ArrayIndexOutOfBoundsExceptionindex past the array/list end
NumberFormatExceptionInteger.parseInt("abc") — not a number
ClassCastExceptiona bad cast (Lesson 17)

Print debugging

System.out.println("here, x = " + x);   // confirm what x really is

Cheap, always available. Print the suspect value just before the crash to confirm your theory.

Null safety with maps

map.getOrDefault(key, fallback)   // value or fallback — no null
if (map.containsKey(key)) { ... } // check before get

Next level: the debugger

VS Code's Java extension: set a breakpoint (click the gutter), run in debug mode, and the program pauses there so you can step line by line and inspect every variable. More powerful than prints once a bug is subtle.
