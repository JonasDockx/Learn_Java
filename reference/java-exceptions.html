Exceptions: try / catch

Survive a thrown error instead of crashing. Print this.

The shape

try {
    riskyThing();              // might throw
    System.out.println("ok");  // skipped if it throws
} catch (IllegalArgumentException e) {
    System.out.println("handled: " + e.getMessage());
}
// execution continues here either way

Parts

PartMeaning
try { }attempt code that might throw
catch (Type e)run this if that type was thrown
e.getMessage()the message given when thrown
throw new ...raise an exception yourself

Python → Java

PythonJava
try:try { }
except T as e:catch (T e) { }
raise T("msg")throw new T("msg");

Why it matters

An uncaught exception stops the program — every later line is skipped. catch turns that crash into a handled case so execution continues. Essential once a program must keep serving (web servers, anything long-running).

Reading a stack trace

at Book.validatePages(Library.java:35)   ← where it was thrown
at Book.<init>(Library.java:20)          ← who called that (<init> = constructor)
at Library.main(Library.java:3)          ← where it started

Top line = where it broke. The exception's message = what went wrong. Scan for the first line number in your file.
