Generics

What <> means: a type placeholder filled in by the caller. Print this.

Using generics (the common case)

ArrayList<String> list = new ArrayList<>();   // a list of String
HashMap<String, Integer> map = new HashMap<>(); // key String, value Integer

The type in <> fills the class's placeholder, so the compiler enforces the contents and you never cast.

Writing a generic class

class Box<T> {            // T = placeholder for a type chosen later
    private T item;
    public void put(T item) { this.item = item; }
    public T get() { return item; }
}

Box<String>  s = new Box<>();   // every T becomes String
Box<Integer> n = new Box<>();   // every T becomes Integer

Placeholder names (convention)

LetterUsually
TType
EElement (List<E>)
K, VKey, Value (Map<K,V>)

Why generics

One class, many types, type-safe. Write Box<T> once instead of StringBox/IntBox; the compiler rejects the wrong type at compile time and removes the need for casts. Generics are how static typing reaches inside containers.

Python comparison

Python containers hold anything; type hints (list[str]) are optional and unenforced. Java's <> is enforced by the compiler.

Not yet covered
