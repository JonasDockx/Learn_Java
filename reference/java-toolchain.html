The Java Toolchain

Quick reference for the compile-and-run pipeline. Print this.

The pipeline

  Foo.java  ──[ javac Foo.java ]──▶  Foo.class  ──[ java Foo ]──▶  output
  source           compile             bytecode        run (JVM)

The three tools

ToolWhat it isYou use it to
JDKJava Development KitThe whole bundle — compiler + runtime + tools. Install this.
javacThe compilerTurn .java source into .class bytecode.
javaThe launcher (JVM)Execute a compiled class's main method.

Commands you'll actually type

java -version          # check the runtime is installed
javac -version         # check the compiler is installed

javac Hello.java       # compile  → Hello.class
java Hello             # run      → (no .class extension!)

Rules to remember

  • A public class must live in a file of the same name: public class Hello → Hello.java.
  • Compile with the extension (Hello.java); run without it (Hello).
  • Every runnable program needs public static void main(String[] args) — the entry point.
  • Statements end in ;. Blocks use { }.

Mental model

javac is the gatekeeper — it won't emit bytecode if your types don't check out. java is the engine — it runs the bytecode the same way on any OS. Source is for humans, bytecode is for the JVM.
