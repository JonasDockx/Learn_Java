# Java Resources

Curated, high-trust sources for the junior-Java-developer mission. Explainers
draw knowledge from here — not from parametric guesses.

## Knowledge

- [The Java™ Tutorials — Oracle](https://docs.oracle.com/javase/tutorial/)
  The official language + standard-library tutorials, written by the people who make Java. Use for: authoritative reference on language features and core APIs.
- [dev.java — Oracle & the Java community](https://dev.java/learn/)
  The modern, actively maintained learning hub (newer than the classic tutorials). Use for: up-to-date language features (records, sealed classes, switch expressions, modern streams).
- [Baeldung](https://www.baeldung.com/)
  Clear, example-driven articles on Java and especially Spring/Spring Boot. Use for: practical "how do I do X" with runnable snippets, enterprise ecosystem topics.
- [_Effective Java_ (3rd ed.) — Joshua Bloch](https://www.oreilly.com/library/view/effective-java-3rd/9780134686097/)
  The canonical book on writing idiomatic, robust Java. Use for: once the basics land — best practices, the "why" behind clean Java. Buy it.
- [Spring Boot Reference Documentation](https://docs.spring.io/spring-boot/index.html)
  Official Spring Boot docs and guides. Use for: when we reach build tooling, dependency injection, REST, and persistence.
- [Maven — Getting Started Guide](https://maven.apache.org/guides/getting-started/)
  Official Maven docs. Use for: project layout, `pom.xml`, dependencies, the build lifecycle.
- [JUnit 5 User Guide](https://junit.org/junit5/docs/current/user-guide/)
  Official JUnit 5 docs. Use for: writing tests, assertions, annotations — the basis of TDD.
- [_Pro Git_ (2nd ed.) — Chacon & Straub (free online)](https://git-scm.com/book/en/v2)
  The canonical, free Git book. Use for: anything Git — the early chapters (Getting Started, Git Basics, Branching) cover everything a junior needs.

## Wisdom (Communities)

- [r/learnjava](https://www.reddit.com/r/learnjava/)
  Beginner-friendly, well-moderated. Use for: "is this idiomatic?", code critique, getting unstuck.
- [Stack Overflow — \[java\] tag](https://stackoverflow.com/questions/tagged/java)
  Use for: specific, well-scoped technical questions once you've tried to solve it yourself.
