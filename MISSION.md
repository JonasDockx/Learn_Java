# Mission: Java for a Junior Developer Role

## Why
Jonas starts a junior Java developer job in a few months and wants to walk in
already fluent — not scrambling to keep up. The goal is to be the junior who
ships clean, working code and understands *why* the team does things the way
they do, rather than copy-pasting from Stack Overflow.

## Success looks like
- Reads and writes idiomatic Java (classes, generics, streams, exceptions) without reaching for a reference for the basics
- Can take a small feature from a ticket to a tested, refactored, committed change using Git + TDD
- Understands the Spring Boot request → service → repository → database flow well enough to add an endpoint
- Can explain core OO and design-principle decisions out loud in a code review
- Has shipped at least one full-stack slice (REST API + persistence + a simple frontend)

## Constraints
- Has a real start date a few months out — pace toward job-readiness, not academic completeness
- Already an intermediate Python programmer; this is about Java + the enterprise ecosystem, not "what is a variable". Frame new concepts against Python, not JavaScript.
- Learns by doing — lessons should be short, hands-on, and produce something runnable

## Out of scope (for now)
- Deep JVM internals (GC tuning, bytecode hacking) beyond a working mental model
- Advanced frontend (Angular comes late; HTML/CSS/JS only as needed for full-stack slices)
- Kubernetes / cloud ops / microservices architecture — past junior scope
