ArrayList — a Resizable List

One idea: a list that grows and shrinks, unlike a fixed-size array.

Your Describable[] shelf had its size fixed the moment you created it — you can't add a book later. An ArrayList is the everyday answer: add and remove items freely. This is the closest thing to a Python list, and you'll use it constantly.

The problem with arrays

String[] names = new String[2];   // exactly 2 slots, forever
names[0] = "Ann";
names[1] = "Bob";
// names[2] = "Cy";   // ✗ crash — no third slot

An array's length is locked at creation. When you don't know how many items you'll have, you need something that grows.

ArrayList — straight from the Python list

# Python                       // Java
import = (built in)            import java.util.ArrayList;   // needed: it's in java.util

names = []                     ArrayList<String> names = new ArrayList<>();
names.append("Ann")            names.add("Ann");
names.append("Bob")            names.add("Bob");
names[0]                       names.get(0);        // "Ann"
len(names)                     names.size();        // 2
names.remove("Ann")           names.remove("Ann");

Three things to notice

  1. import java.util.ArrayList; — ArrayList lives in another package (java.util), so you must import it. (Remember: same-package classes need no import; this one isn't.)
  2. <String> — the type of thing the list holds. ArrayList<String> can hold only Strings; static typing applies to the contents too. (The full story of <> is "generics", a later lesson — for now: "a list of Strings".)
  3. Methods, not operators — Java uses .add(), .get(i), .size() where Python uses append, [i], and len().

The whole thing in one file

import java.util.ArrayList;

public class ListIntro {
    public static void main(String[] args) {
        ArrayList<String> names = new ArrayList<>();

        names.add("Ann");
        names.add("Bob");
        names.add("Cy");          // grows freely — no fixed size
        names.remove("Bob");      // shrinks too

        System.out.println("count: " + names.size());   // 2

        for (String name : names) {     // same for-each you already know
            System.out.println(name);
        }
    }
}

Compile and run → count: 2, then Ann and Cy.

Check yourself

Why must you write import java.util.ArrayList;?
What does <String> in ArrayList<String> mean?
Python's names.append("x") becomes which Java call?

Do this now (one file, small steps)

  1. Create ListIntro.java with the code above. Compile and run — confirm count: 2 and the two names.
  2. Forget the import on purpose: delete the import line and recompile. Read the "cannot find symbol: class ArrayList" error, then restore it.
  3. Add two more names, remove one, and print names.size() and the list. Confirm the count matches.
  4. Try names.add(42); — read the compile error. The list is <String>, so a number is rejected. That's static typing on the contents.
