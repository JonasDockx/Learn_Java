Debugging

One skill: read the crash, find the line, inspect the value, fix it.

Bugs are the job. The difference between a stuck junior and a productive one isn't avoiding bugs — it's a calm, repeatable way to find them. You'll practise that loop on a real, common crash: the NullPointerException.

The loop

  1. Reproduce — run it, get the error.
  2. Read the stack trace — the top line + message say where and what.
  3. Inspect — print the suspicious values to see what's actually there.
  4. Fix and verify — change one thing, run again.

A bug to hunt

Put this in BugHunt.java and run it:

import java.util.HashMap;

public class BugHunt {
    public static void main(String[] args) {
        HashMap<String, Integer> ages = new HashMap<>();
        ages.put("Ann", 30);

        String name = "Bob";              // not in the map
        int age = ages.get(name);         // ← crashes here
        System.out.println(name + " is " + age);
    }
}

It throws something like:

Exception in thread "main" java.lang.NullPointerException:
  Cannot invoke "java.lang.Integer.intValue()" because the return value of
  "java.util.HashMap.get(Object)" is null
        at BugHunt.main(BugHunt.java:9)

Read it

  • What: NullPointerException — code tried to use something that was null (the "arrow pointing at nothing" from Lesson 4).
  • Where: BugHunt.java:9 — the int age = ages.get(name); line.
  • Why (the message spells it out): HashMap.get(...) returned null (no "Bob" key — the Lesson 16 gotcha), and Java can't turn null into an int.

Modern Java's "helpful NullPointerException" names the exact thing that was null — read that message; it's doing half your work.

Inspect with a print

When the cause isn't obvious, print the suspect before the crashing line:

System.out.println("got from map: " + ages.get(name));   // prints: got from map: null
int age = ages.get(name);

Seeing null confirms it. (Note: store it as Integer to print without crashing — converting to int is what fails.)

The fix — guard the null

Don't assume the key exists. Two clean options: 
// Option A: a default value
int age = ages.getOrDefault(name, 0);     // 0 if absent — no crash

// Option B: check first
if (ages.containsKey(name)) {
    int age = ages.get(name);
    System.out.println(name + " is " + age);
} else {
    System.out.println(name + " not found");
}

Check yourself

A NullPointerException means the code…
In a stack trace, where is the crash in your code?
What does ages.getOrDefault("Bob", 0) do?

Do this now (one file, small steps)

  1. Create BugHunt.java above and run it. Read the NullPointerException — find the line number and the "is null" message.
  2. Add the inspect print (storing the result as Integer) and confirm it shows null.
  3. Fix it with getOrDefault(name, 0), run, and confirm it prints Bob is 0 with no crash.
  4. Change name to "Ann" and confirm it prints Ann is 30.
