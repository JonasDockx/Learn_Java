Exception Handling: try / catch

One idea: catch an error so the program keeps running instead of crashing.

You just watched Library die on an uncaught IllegalArgumentException — the program stopped and nothing after it ran. This lesson is the tool that prevents that: wrap the risky code, handle the problem, carry on.

The problem

When code throws an exception and nobody catches it, the program stops immediately — every line after the failure is skipped, and you get a stack trace. That's fine for a tiny script, but a real program (a web server handling requests) must not die because one input was bad.

The shape — straight from Python

You know this structure from Python; Java renames the keywords:

# Python                          // Java
try:                              try {
    check_age(age)                    checkAge(age);
except ValueError as e:           } catch (IllegalArgumentException e) {
    print("bad:", e)                  System.out.println("bad: " + e.getMessage());
                                  }

What it means

  • try { ... } — "attempt this; it might throw."
  • catch (IllegalArgumentException e) { ... } — "if it threw this kind of exception, run this instead of crashing." e is the exception object.
  • e.getMessage() — the text that was put in the exception when it was thrown (your "age cannot be negative").

The whole thing in one file

Put this in TryCatchIntro.java:

public class TryCatchIntro {
    public static void main(String[] args) {
        int[] ages = {25, -3, 40};

        for (int age : ages) {
            try {
                checkAge(age);
                System.out.println(age + " is valid");
            } catch (IllegalArgumentException e) {
                System.out.println("Skipped: " + e.getMessage());
            }
        }

        System.out.println("Program finished normally");   // this STILL runs
    }

    static void checkAge(int age) {
        if (age < 0) {
            throw new IllegalArgumentException("age cannot be negative: " + age);
        }
    }
}

Compile and run. Output:

25 is valid
Skipped: age cannot be negative: -3
40 is valid
Program finished normally

The key insight

The bad value (-3) threw, catch handled it, and the loop kept going to 40 — then the final line ran too. Catching converts a fatal crash into a handled bump in the road. Without the try/catch, the program would stop at -3 and never reach 40 or "finished".

Do this now (one file, small steps)

  1. Create TryCatchIntro.java with the code above. Compile and run — confirm all four output lines, including "finished".
  2. Remove the safety net: temporarily replace the whole try/catch with just checkAge(age); and System.out.println(age + " is valid");. Run it — watch it crash at -3; 40 and "finished" never print. Then put the try/catch back.
  3. Add another bad age (e.g. -99) to the array and confirm both are skipped and the program still finishes.
