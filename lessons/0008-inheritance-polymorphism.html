Inheritance & Polymorphism

Let Dog and Cat share what they have in common — then treat them the same while they behave differently.

Two pillars at once. Dog and Cat both have a name and an age and can make a sound. Copy-pasting that into every animal is exactly the duplication OO exists to kill. Inheritance shares it; polymorphism lets one piece of code work with all of them. This pairing is the backbone of how Java frameworks (Spring included) are built.

Inheritance — a shared parent

Pull the common parts into an Animal parent; Dog and Cat extend it and add their own twist.

public class Animal {
    private String name;

    public Animal(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }

    public String speak() {       // a default; subclasses will override
        return name + " makes a sound";
    }
}
public class Dog extends Animal {     // Dog IS-A Animal
    public Dog(String name) {
        super(name);                  // call Animal's constructor
    }

    @Override
    public String speak() {           // replace the inherited behaviour
        return getName() + " says woof";
    }
}
public class Cat extends Animal {
    public Cat(String name) {
        super(name);
    }

    @Override
    public String speak() {
        return getName() + " says meow";
    }
}
  • extends Animal — Dog inherits Animal's fields and methods (the "is-a" test: a Dog is an Animal).
  • super(name) — calls the parent constructor to set up the inherited part. (Python: super().__init__(name).)
  • @Override — tells the compiler "I'm deliberately replacing the parent's speak()." If you typo the name, the compiler catches it.
  • getName() — Dog can't touch private name directly (per-class privacy, Lesson 7!), so it uses the inherited public getter.

Polymorphism — one type, many forms

Because a Dog is an Animal, an Animal variable can hold one. And when you call speak(), Java runs the actual object's version — decided at runtime:

Animal a = new Dog("Rex");
Animal b = new Cat("Mittens");

System.out.println(a.speak());   // Rex says woof   ← Dog's version
System.out.println(b.speak());   // Mittens says meow ← Cat's version

// The payoff: one loop, any animal
Animal[] zoo = { new Dog("Rex"), new Cat("Mittens"), new Animal("Thing") };
for (Animal animal : zoo) {
    System.out.println(animal.speak());   // each speaks for itself
}

The key insight

The variable's declared type is Animal, but the object's real type decides which speak() runs. This is polymorphism: write code against the general type (Animal), and every specific type plugs in and behaves correctly. Add a Hamster tomorrow and that loop needs zero changes. (This is Python's duck typing — but Java reaches it through a shared supertype the compiler checks.)
