Streams

One idea: process a collection as a pipeline — filter, transform, collect.

This is where lambdas earn their keep. A stream lets you take a list, keep the items you want, transform them, and gather the result — as a readable chain instead of a manual loop. It's the Java answer to Python's list comprehensions, and it's all over modern codebases.

The shape: source → operations → result

var result = nums.stream()          // 1. start a pipeline from the list
    .filter(n -> n % 2 == 0)         // 2. keep only evens     (a lambda)
    .map(n -> n * 10)                // 3. transform each       (a lambda)
    .toList();                       // 4. gather into a new list

Three kinds of step:

  • stream() — turns the collection into a pipeline.
  • Intermediate ops (filter, map) — each takes a lambda and returns a new stream, so they chain. filter keeps items; map transforms them.
  • Terminal op (toList, forEach) — ends the pipeline and produces the result. Nothing runs until a terminal op is reached.

The Python comparison

# Python list comprehension
result = [n * 10 for n in nums if n % 2 == 0]

// Java stream
var result = nums.stream()
    .filter(n -> n % 2 == 0)
    .map(n -> n * 10)
    .toList();

Same three ideas — source, a condition to keep (if / filter), a transform (n*10 / map). Python packs it in one expression; Java spells each step as a named link in a chain.

The key insight

Each filter/map takes a lambda — exactly the x -> ... from Lesson 18. A stream is just a sequence of lambdas applied to a collection. And it does not change the original list — it produces a new result, leaving nums untouched.

The whole thing in one file

import java.util.ArrayList;

public class StreamIntro {
    public static void main(String[] args) {
        ArrayList<Integer> nums = new ArrayList<>();
        nums.add(1); nums.add(2); nums.add(3); nums.add(4);

        var result = nums.stream()
            .filter(n -> n % 2 == 0)     // 2, 4
            .map(n -> n * 10)            // 20, 40
            .toList();

        System.out.println(result);      // [20, 40]
        System.out.println(nums);        // [1, 2, 3, 4] — unchanged

        // A terminal op that acts instead of collecting:
        nums.stream()
            .filter(n -> n > 2)
            .forEach(n -> System.out.println("big: " + n));   // big: 3, big: 4
    }
}

% is the remainder operator: n % 2 == 0 means "evenly divisible by 2" — even.
