Interfaces

One idea: a contract that a class promises to fulfil.

The second half of abstraction, on its own. You just saw abstract classes; an interface is the next step along the same line. One concept, one file, fresh names.

The idea

An interface is a list of method signatures with no bodies and no state — a pure contract. A class implements the interface and must provide every method, or it won't compile.

Compared to the abstract class from Lesson 11:

Abstract classInterface
can have fields + finished methodsno fields, no bodies — just the contract
a class extends onea class implements many

One file, two pieces

Put this in InterfaceIntro.java:

public class InterfaceIntro {
    public static void main(String[] args) {
        Switchable lamp = new Lamp();      // a Lamp IS-A Switchable
        System.out.println(lamp.turnOn());
    }
}

interface Switchable {          // the contract
    String turnOn();            // no body — implementers must provide it
}

class Lamp implements Switchable {   // signs the contract
    @Override
    public String turnOn() {         // must be public (interface methods are public)
        return "Lamp is on";
    }
}

Compile and run:

javac InterfaceIntro.java
java InterfaceIntro

Prints Lamp is on. Lamp compiles because it fulfils the Switchable contract.

Two things to notice

  1. The implementing method must be public. Interface methods are always public, and a class can't make them less visible.
  2. The variable's type can be the interface: Switchable lamp = new Lamp();. You program to the contract, not the concrete class — the same polymorphism idea from Lesson 8.

See it fail on purpose

Delete the turnOn() method from Lamp and recompile (clean build if needed — remember Lesson 11's stale .class trap):

error: Lamp is not abstract and does not override abstract method turnOn() in Switchable

The contract is unfulfilled, so the class won't compile. Put the method back.

Check yourself

What can an interface contain?
How many interfaces can one class implement?
Why must Lamp.turnOn() be public?

Do this now (one file, small steps)

  1. Create InterfaceIntro.java with the code above. Compile and run — confirm Lamp is on.
  2. Delete turnOn() from Lamp, recompile, read the "does not override" error, then restore it.
  3. Add a second class Fan implements Switchable whose turnOn() returns "Fan is spinning". Create one in main and print it.

That is the whole lesson: an interface is a contract; a class that implements it must provide every method.
