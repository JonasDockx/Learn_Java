HashMap — Look Up by Key

One idea: store values under a key and fetch them back by that key.

A list finds things by position (get(0)). Often you want to find by a name instead — "what's Ann's age?" A HashMap does that: it stores key → value pairs. It's Python's dict, and it's everywhere in real code.

HashMap — straight from the Python dict

Two things to notice

  1. Two types in the <>: HashMap<String, Integer> means keys are String, values are Integer. Key type first, value type second.
  2. Integer, not int: collections hold objects, and Integer is the object form of the primitive int. Java converts between them automatically, so you still write ages.put("Ann", 30). (Why there are two forms is a later detail — for now: in collections, use Integer.)

The whole thing in one file

import java.util.HashMap;

public class MapIntro {
    public static void main(String[] args) {
        HashMap<String, Integer> ages = new HashMap<>();

        ages.put("Ann", 30);
        ages.put("Bob", 25);
        ages.put("Ann", 31);          // same key overwrites — now Ann is 31

        System.out.println("Ann is " + ages.get("Ann"));       // 31
        System.out.println("Has Cy? " + ages.containsKey("Cy")); // false
        System.out.println("count: " + ages.size());            // 2

        for (String name : ages.keySet()) {     // loop over the keys
            System.out.println(name + " -> " + ages.get(name));
        }
    }
}

Compile and run. Note put with an existing key overwrites the old value — same as a Python dict.

One gotcha: missing keys return null

ages.get("Nobody")   // returns null (not a crash) — there's no such key

Where Python raises KeyError, Java hands back null (an arrow pointing at nothing — Lesson 4). Use containsKey first when you're unsure.
