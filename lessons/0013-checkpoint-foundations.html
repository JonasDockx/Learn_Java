Checkpoint: Foundations Review

No new material. Three ways to test what's stuck: explain it, recall it, build with it.

This reviews Lessons 1–12. Work top to bottom, but stop whenever you want — the goal is to find which ideas are solid and which need another pass, not to finish in one sitting.

Part 1 — Explain it yourself

Answer these in the chat, in your own words. Explaining out loud is the strongest test of real understanding — I'll tell you where you're exactly right and where to sharpen.

  1. Trace what happens from saving Hello.java to seeing output. Use the words javac, .class, and JVM.
  2. Why does == sometimes give the wrong answer when comparing String text, and what should you use instead?
  3. What does static mean on main, and how is that different from a method like speak() on a Dog?
  4. What does private protect, and from whom? (Think about the per-class vs per-object point.)
  5. In one or two sentences each: how is an abstract class different from an interface?

Part 3 — Small project (build with it)

This recombines patterns you've already written: an interface (like Switchable), encapsulation (like Dog's validating setter), and a loop over an array (like Grades). Nothing new — just assembled together. Build it in a single file, Library.java.

Step by step

  1. The contract. Write an interface Describable with one method: String describe();
  2. The class. Write class Book implements Describable with private String title; and private int pages;, a constructor, and getters. In the constructor, reject pages < 1 with an IllegalArgumentException (encapsulation).
  3. Fulfil the contract. Implement describe() to return something like title + " (" + pages + " pages)".
  4. Use it. In public class Library's main, make an array Describable[] shelf = { new Book("Java", 300), new Book("Git", 120) }; and loop over it printing describe() for each.
  5. Prove the guard. Try new Book("Bad", 0) and run — watch your exception fire.
  6. Commit it on a branch and merge (Git practice).

Why this is the real test

If you can build this without copying from earlier files, you've genuinely integrated the core: classes, objects, private, an interface, an array, a loop, and the compile/run cycle — all at once. That integration is what "knowing Java" actually means at this stage.
